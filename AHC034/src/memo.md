# AHC034 memo

## 問題概要

- ダンプカーで平坦な土地を作成する問題

- 最終的なコストが少し複雑

  - base は元の土地の絶対値総和
  - cost は移動・運搬・堆積のコスト
  - diff は最終的な土地の絶対値 \* 100 + 10000

- diff を 0 にすることのみ考えると、逆に cost が膨張するということもあるかもしれない
- cost と diff の均衡をうまく保つ問題？

- 入力の傾斜は結構なだらか（パーリンノイズ）

## 考察

- 無駄に往復するくらいなら、一気に運ぶ方が良い
- 巡回する移動パターンを色々用意して、正だったら積む・負だったら降ろすという方針はあり
- 結局 All 0 にするのを一度試したいので、正 ⇒ 負の順で訪れる感じでやるのもあり

  - ただ負に行った後でまだ堆積が残る場合はまだ負に行く必要がある

- 最終的な盤面コストが大切なだけで、途中にすごい凸凹になっても別に良い
  ⇒ それを利用した貪欲で All 0 かつ Cost 200,000 ~ 300,000 位
- 上記貪欲は移動が 40000 ~ 80000 なので、出来るだけ運搬・堆積を減らす方向で考える
- まだ良い貪欲がありそう

- 方向転換を加えて 8 通りで一番良いものを出したら平均 120,000 くらいになった

- これって別にジグザグ系でなくても良いのでは？
- 貪欲部分をスコアにしてビームサーチをすれば良い？
  - 2 方向近傍
  - 次に行ける方向と score と map を状態に持つ
