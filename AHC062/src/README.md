# A - King's Tour

## 問題概要

AtCoder王国は $N \times N$ のグリッド状の区画に区切られており、各区画にはそれぞれ異なる人数の国民が住んでいます。国王の高橋君は、縦・横・斜めに隣接する区画を辿りながら、すべての区画を一度ずつ訪問するツアーを計画しています。ツアーは日を追うごとに盛り上がっていくため、後の日に訪れた区画ほど、住人の高橋君への好感度が高くなります。高橋君が得る好感度の総和がなるべく大きくなるようなツアーを計画してください。

## 制約
- $N = 200$（固定）
- 各区画 $(i, j)$ の住民数 $A_{i,j}$ は $1 \leq A_{i,j} \leq N^2$
- すべての区画を一度ずつ訪問する
- 連続する2日で訪問する区画は縦・横・斜めに隣接している必要がある
- ツアーの開始区画・終了区画は任意
- 実行時間制限: 3秒
- メモリ制限: 1024 MiB

## 入力形式
```
N
A_{0,0} A_{0,1} ... A_{0,N-1}
...
A_{N-1,0} ... A_{N-1,N-1}
```

## 出力形式
ツアーの $k$ 日目に訪問する区画 $(i_k, j_k)$ を以下の形式で出力してください。
```
i_0 j_0
i_1 j_1
...
i_{N^2-1} j_{N^2-1}
```

- $0 \leq i_k, j_k \leq N-1$
- $(i_k, j_k)$ はすべて異なる
- $\max(|i_k - i_{k+1}|, |j_k - j_{k+1}|) = 1$（隣接）

## スコア計算
$k$ 日目に訪問した区画 $(i_k, j_k)$ で住民1人あたり $k$ の好感度が得られる。高橋君が得る好感度の総和 $V$ は
$$
V = \sum_{k=0}^{N^2-1} k \cdot A_{i_k, j_k}
$$

得点は $\mathrm{round}\left(\frac{V}{N^2}\right)$ となる。

## 入力生成方法
- $N=200$ 固定
- $1$ 以上 $N^2$ 以下の整数を一様ランダムに並び替え、順に $A_{0,0}, A_{0,1}, ..., A_{N-1,N-1}$ とする

## 注意事項
- 合計100ケースの合計得点が提出の得点となる
- 不正な出力や制限時間超過が1ケースでもあると全体が無効
- コンテスト中の最高得点で順位決定、終了後のシステムテストなし
- 解法・考察の公開は禁止
