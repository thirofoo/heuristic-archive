# AHC063 - Colorful Ouroboros

## 問題概要

N×N マスの盤面上で蛇を操作する最適化問題。

### 初期状態
- 蛇の長さ: 5
- 初期位置: `(4,0),(3,0),(2,0),(1,0),(0,0)`（頭→しっぽ順）
- 初期色: すべて 1
- 盤面上に M-5 個の餌（色 1〜C）が散乱

### 操作ルール（1ターン）
1. **移動**: 上下左右に1マス移動（Uターン不可・盤面外不可）
2. **食事**: 移動先に餌がある場合、蛇が1マス伸び、その色が**しっぽに追加**される
3. **噛みちぎり**: 頭が胴体と同じ座標になると、その位置以降のしっぽが切れて餌として復元される（食事と同時発生なし）

### 色の仕様
- 色列 `c[0],...,c[k-1]` は蛇の論理インデックスに紐付く（ヘッド=0、テール=k-1）
- 移動だけでは色列は変化しない
- 餌を食べると食べた色が末尾に追加される
- 噛みちぎりが起きると末尾の色が削除される

### スコア（最小化）
```
T + 10000 × (E + 2(M-k))
```
- T: 操作数
- k: 最終的な蛇の長さ
- E: 色列 `c[0],...,c[k-1]` のうち好み `d[0],...,d[k-1]` と異なる位置の個数
- `d[0]...d[4] = 1`（初期色と一致）

### 入力
```
N M C
d[0] ... d[M-1]       # 好みの色列
f[0][0] ... f[0][N-1] # 盤面（0=餌なし、1〜C=餌の色）
...
f[N-1][0] ... f[N-1][N-1]
```
- 8 ≤ N ≤ 16
- N²/4 ≤ M ≤ 3N²/4
- 3 ≤ C ≤ 7
- 最大 100000 ターン

---

## 解法方針

### 貪欲 + BFS アプローチ

**基本戦略**: 好みの色列 `d[5], d[6], ..., d[M-1]` の順に、対応する色の餌を探して食べる。

**アルゴリズム**:
1. 次に食べたい色 `targetColor = desired[p]` を決定
2. 盤面上の `targetColor` の餌の中で、頭からのマンハッタン距離が最小のものを選択
3. BFS で頭から目標餌まで最短経路を探索（**他の餌を避けながら**）
4. 経路を実行し、目標を食べたら `p++`

**BFS の優先順**:
- 1st: `avoidFood=true, avoidBody=true`（最安全）
- 2nd: `avoidFood=true, avoidBody=false`（蛇の体を貫通、噛みちぎり可能性あり）

**噛みちぎり検出**: 実行中に蛇の長さが減少した場合（噛みちぎり発生）、`p = body.size()` にリセットして再計画。

**行き詰まり対策**: 経路が見つからない場合、餌を避けながらランダムに1手動かして状況を変える。

### 期待される効果
- T ≈ M × (平均移動距離)、M ≤ 192、距離 ≤ 32 → T ≈ 6000
- E = 0（正しい順序で食べれば）
- k = M（全ての餌を食べれば）
- スコア ≈ 6000
