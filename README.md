# A - Ore Rolling (A)

## 問題概要

- 20 × 20 のグリッドで表現される洞窟内に散らばる岩と鉱石を、指定された穴に落とすための操作シーケンスを求める
- 最大 10,000 回の操作で、各種類の鉱石を対応する穴へできるだけ効率的に落とす
- 岩はどの穴に落としてもよく、洞窟内に残しても問題無し

## 洞窟の構成

- **グリッド:**
  - サイズは N×N（全テストケースで N = 20 固定）
  - 各マスの座標は、左上を (0,0) とし、下に i マス、右に j マス進んだ (i,j) で表現

- **オブジェクトの種類:**
  - **岩 (`@`):** どの穴に落としてもよく、洞窟内に放置しても構わない
  - **鉱石 (`a`-`z`):** 対応する穴（大文字 `A`-`Z`）に落とす必要がある
  - **穴 (`A`-`Z`):** 各種類の鉱石に対して、必ず 1 つだけ存在する

- **初期位置:**
  - プレイヤーは最初、穴 `A` のマスに配置される

## 操作の種類

1. **移動 (操作番号: 1):**
   - 上下左右の隣接セルへ移動
   - 移動先に穴、岩、鉱石があっても移動可能

2. **運ぶ (操作番号: 2):**
   - 現在のセルにある岩または鉱石を、上下左右の隣接セルへ運ぶ
   - 運ぶ先のセルに他の岩・鉱石が存在してはいけない
   - 運ぶ先が穴の場合、対象は穴に落ちて除去
   - 操作後、プレイヤーの位置は運んだ先のセルになる

3. **転がす (操作番号: 3):**
   - 現在のセルにある岩または鉱石を、上下左右のいずれかの方向に転がす
   - 転がすと対象は、次のいずれかの条件が成立するまで一直線に移動する
     - **穴に落ちる:** 転がしている方向の隣接セルが穴の場合、対象は穴に落ち、除去
     - **衝突して停止:** 転がしている方向の隣接セルに岩または鉱石がある、またはグリッド外に出る場合、対象は現在のセルで停止
     - **連続移動:** 上記に該当しなければ、対象は隣接セルへ移動し、同様の処理が繰り返す。
   - 転がす動作は非常に速いため、次の操作を行う前に必ず対象は停止するか穴に落ちる

## 問題の目的と得点計算

### 目的

- 基本全部正しく落とし、そのターンを最小化する

## 問題制約

### A 問題

- 鉱石 1 種類固定
- 岩や穴が比較的少ない
    - 穴は 1 個のみ
    - 鉱石・岩は 2 N 個

### B 問題

- 鉱石 3 種類固定
- 穴が比較的少ない
    - 穴は 3 個のみ
    - 鉱石はそれぞれ N 個ずつ
    - **岩は無し**

### C 問題

- 鉱石 1 種類固定
- 岩や穴が比較的多い
    - 穴は 1 個のみ
    - 鉱石は 2 N 個
    - 岩は N*N / 2 個
